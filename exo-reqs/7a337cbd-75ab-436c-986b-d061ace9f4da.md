# req(exo): validate schema does NOT emit a false sh:class violation when a value is typed by a label-less UUID-named class that is a subclass of the property range

## Job Story

When I validate a vault that uses a label-less UUID-named class (e.g. period__Year, declared only by its superClass) as the type of a referenced asset, I want the SHACL sh:class range check to recognise that class's subclass chain and NOT raise a false violation, so that valid subclass-chain references aren't flagged as schema errors (issue #3242).

## Statement (Gherkin)

```gherkin
Given a class file period__Year that is UUID-named, has NO exo__Asset_label, and
  declares exo__Class_superClass reaching exo__Asset through the chain
And an asset year-2026 typed exo__Instance_class period__Year
And a reference asset whose exo__Asset_relates points at year-2026
  (the exo__Asset_relates shape has range exo__Asset)
When I run `validate schema --shapes-mode` over the vault
Then no sh:class violation is emitted against exo__Asset on the exo__Asset_relates path
And the label-less class chain resolves year-2026 as conforming to the exo__Asset range
```
