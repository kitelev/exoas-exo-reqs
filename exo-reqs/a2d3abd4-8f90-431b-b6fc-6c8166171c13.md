# req(exo): Create Subclass creates a UID-named exo__Class co-located in its isDefinedBy ontology folder (never vault-root classes/<label>.md)

## Job Story

When I click **Create Subclass** on a class page, I want the new `exo__Class` to land in the folder of its `exo__Asset_isDefinedBy` ontology and be named after its own UID — exactly like Create Instance already does — so that the new TBox class obeys the CO-LOCATION INVARIANT and UUID-CANON TBOX, resolves its symbolic IRI / Properties block, and never silently scatters into a vault-root `classes/` folder that the resolver and SHACL can't reach.

## Statement (Gherkin)

```gherkin
Given a class C with exo__Asset_isDefinedBy = $X, on the class page of C
When I run "Create Subclass" with class name N
Then the new exo__Class is UID-named "<new-uid>.md" (NOT label-named "<n>.md")
And it is co-located in the folder where $X's ontology file lives
  (e.g. isDefinedBy=$exo [[ca97bb2f]] -> assetspaces/kitelev/exoas-exo/exo/)
And it is NOT created in vault-root "classes/<label>.md"
And when $X is unresolvable the new class falls back to C's own (co-located) folder, still UID-named
```
