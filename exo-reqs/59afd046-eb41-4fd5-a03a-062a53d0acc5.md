# req(exo): on mobile, Add AssetSpace materialises at the canonical Maven path and appends it as a second .gitmodules entry (Desktop↔Mobile parity)

## Job Story

When I add a further AssetSpace on iPhone (no git binary), I want it materialised via vault.adapter at the SAME canonical Maven path that bootstrap and apply-profile use, so a later apply-profile recognises it as already-mounted instead of re-materialising the same repo at a different path (double mount) — the Desktop↔Mobile Command Parity invariant.

## Statement (Gherkin)

```gherkin
Given a mobile vault already bootstrapped exo-only (one .gitmodules entry)
When I run «Exocortex: Add AssetSpace by URL» for a second AssetSpace
Then it is materialised via vault.adapter at the Maven path assetspaces/<owner>/<repo> (#3538)
And `.gitmodules` lists exactly [exo (Maven), the second AssetSpace (Maven)] in order
And an "AssetSpace added" notice is shown
```
