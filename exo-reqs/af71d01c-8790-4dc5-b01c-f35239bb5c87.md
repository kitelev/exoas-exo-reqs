# req(exo): obsplugin__Setting export — Periodic Notes pilot (RFC f402002b, M4.1)

## Job Story

When I want to distribute my Periodic Notes Daily-Notes configuration (folder/format/template) across devices or to a fresh tester — instead of hand-rebuilding it per device — I run «Exocortex: Export Periodic Notes settings», pick a target ontology, and the plugin snapshots Periodic Notes' live `data.json` into homoiconic `obsplugin__Setting` assets in the graph, reusing the same generic engine that distributes Exocortex's own settings — so plugin settings become first-class, distributable vault data, the same uniform way every source is, and the export runs identically on desktop and iPhone.

## Statement (Gherkin)

```gherkin
Scenario: export each declared Daily-Notes key into an obsplugin__Setting asset
  Given a Periodic Notes data.json with daily.folder "assetspaces/kitelev", daily.format "YYYY-MM-DD [Note]", daily.template "09 Templates/periodic-notes/{{Daily Note}}.md", daily.enabled true
  When «Export Periodic Notes settings» runs and the user picks ontology O
  Then exactly 4 obsplugin__Setting assets are written into O's folder (one per declared Daily key)
  And each asset is typed exo__Instance_class obsplugin__Setting and exo__Asset_isDefinedBy O
  And the daily.folder asset carries setting__Setting_key [[<DailyFolder keyUid>]] and setting__Setting_value "assetspaces/kitelev"
  And the daily.enabled asset carries setting__Setting_value true (a real boolean, per its declared datatype)
  And each asset's basename is the fixed deterministic settingUid (byte-identical across devices)

Scenario: allowlist-by-construction — undeclared live keys never leak
  Given a Periodic Notes data.json that ALSO contains weekly.*, monthly.* and showGettingStartedBanner
  When «Export Periodic Notes settings» runs
  Then ONLY the 4 declared daily.* keys are exported
  And no weekly.* / monthly.* / banner key is ever written to an asset

Scenario: Periodic Notes not installed → notice, no writes (no crash)
  Given .obsidian/plugins/periodic-notes/data.json is absent or unparseable
  When «Export Periodic Notes settings» runs
  Then the user is notified that Periodic Notes was not found
  And no asset is written

Scenario: mobile-safe + parity
  Given the command runs on iPhone (no Node fs)
  When it reads Periodic Notes data.json
  Then it reads via vault.adapter (not Node fs), and the command is registered unconditionally (no Platform.isMobile gate)
```
