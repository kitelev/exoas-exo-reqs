# req(exo): deriveUrl(path) — pure inverse of derivePath; DRY toHttpsGitHubUrl over it (RFC 0005 Phase 0)

## Job Story

When a device-side flow needs an AssetSpace's clone URL but only has its on-disk folder path (`assetspaces/<owner>/<repo>`) — because the descriptor that carries the URL lives *inside* the folder and is gone when the folder is empty/destroyed — I want a pure `deriveUrl(path)` that reconstructs `https://github.com/<owner>/<repo>`, so the URL no longer has to be stored in a `.gitmodules` sidecar (RFC 0005), and the existing duplicated reconstruction in `toHttpsGitHubUrl` collapses to a single source of truth.

## Statement (Gherkin)

```gherkin
Scenario: deriveUrl reconstructs a GitHub clone URL from a canonical mount path
  Given the AssetSpace path-derivation module (AssetSpacePathDeriver)
  When I call deriveUrl("assetspaces/kitelev/exoas-exo")
  Then it returns "https://github.com/kitelev/exoas-exo"

Scenario: deriveUrl is the inverse of derivePath under the GitHub-https invariant
  Given any GitHub clone URL in https / http / ssh-url / scp-like / git:// form,
        with or without a ".git" suffix, embedded credentials, a port, or a trailing slash
  When I compute deriveUrl(derivePath(url))
  Then the result equals the canonical "https://github.com/<owner>/<repo>" form of that URL

Scenario: deriveUrl returns null for non-conforming paths (caller fallback)
  Given a path that is not a canonical "assetspaces/<owner>/<repo>"
        (wrong prefix, fewer than two segments, an out-of-charset segment, or a "."/".." traversal segment)
  When I call deriveUrl(path)
  Then it returns null

Scenario: toHttpsGitHubUrl preserves its contract while delegating to deriveUrl (DRY)
  Given the CLI apply-profile helper toHttpsGitHubUrl(git)
  When I pass any GitHub git URL form
  Then it returns the canonical "https://github.com/<owner>/<repo>" form (unchanged contract)
  And when I pass an un-parseable string it returns that string unchanged
  And it computes the result via deriveUrl (no duplicated path→url reconstruction remains)
```
