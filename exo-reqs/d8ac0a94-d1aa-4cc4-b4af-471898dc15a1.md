# req(exo): statement-asset UID from a crypto-secure source — never Math.random

## Job Story

When I reify a relation in the Property Editor and a fresh `exo__Statement`
asset's UID is minted, I want that UID drawn from a crypto-secure randomness
source on every platform — so the identifier is not predictable and the plugin
carries no `Math.random()` insecure-randomness (CodeQL `js/insecure-randomness`).

## Statement (Gherkin)

```gherkin
Given the plugin mints a new statement-asset UID via generateStatementUid()
When crypto.randomUUID is available (desktop Electron / modern WebView)
Then the UID is crypto.randomUUID() lowercased — a valid RFC-4122 v4 UUID

Given crypto.randomUUID is unavailable but crypto.getRandomValues is (older mobile WebView)
When a new statement-asset UID is minted
Then the UID is built from crypto.getRandomValues() — a valid lowercase v4 UUID
And Math.random() is NEVER called (no js/insecure-randomness)
```

## Verification

**Bound (unit):** `@req:d8ac0a94-d1aa-4cc4-b4af-471898dc15a1` — the fallback-path
test stubs a crypto object with `getRandomValues` but no `randomUUID`, asserts
`getRandomValues` is called, `Math.random` is NOT, and the output matches the v4
UUID shape. Revert-verify: the old `Math.random()` fallback reds this test;
`crypto.getRandomValues` greens it. (al-3773, #3773 / CodeQL alert 313, 2026-06-28.)
