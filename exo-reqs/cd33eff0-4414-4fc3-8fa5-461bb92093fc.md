# req(exo): rename-to-uid --dry-run previews without renaming the file or mutating frontmatter (Issue #3111)

## Job Story

When I want to preview a UID-canon rename before committing to it, I want `rename-to-uid --dry-run` to show me exactly what would change — the rename, label/alias backfill, and inbound-link rewrites — while leaving every file on disk byte-for-byte untouched, so I can review a destructive operation safely before applying it.

## Statement (Gherkin)

```gherkin
Given a markdown note "Note.md" with a uid and label
When I run rename-to-uid on it with dryRun = true
Then the file is NOT renamed (Note.md still exists; <uid>.md does not)
And its content and inode are unchanged
And the output contains "[dry-run]" and "Would rename"
And the output does NOT contain the applied "✅ Renamed to UID format" line
```
