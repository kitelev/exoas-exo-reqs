# req(exo): CLI can set ems__Effort_parent to an explicit UID as a wikilink (set-parent)

## Job Story

When I have just created a project and a task and want to parent the task under that specific project from the CLI (dogfooding mutation parity), I want `apply set-parent <task> --input '{"parent":"<project-uid>"}'` to set `ems__Effort_parent` to a proper wikilink — so I never have to hand-edit frontmatter and the UWI child→parent invariant holds.

## Statement (Gherkin)

```gherkin
Given a task asset and a target parent UID
When I run `apply set-parent <task> --input '{"parent":"<uid>"}'`
Then the task's ems__Effort_parent is set to "[[<uid>]]" (wikilink form, not a bare scalar)
  And the literal "$input.parent" placeholder never persists
When I instead pass a wrongly-named input key (e.g. {"value":...})
Then the apply fails loud and writes nothing (no half-substituted literal)
```
