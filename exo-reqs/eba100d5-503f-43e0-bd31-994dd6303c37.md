# req(exo): mounting a large AssetSpace (>500 files) emits granular install progress — `N of M files (X%)` every 200 files — instead of going silent

## Job Story

When I apply a profile that materializes a large AssetSpace on a slow device (iPhone), I want the apply log to show install progress (`N of M files (X%)`) advancing every 200 files — the same way vault indexing shows `Indexing vault: N of M assets` — so that I can tell materialization is actively progressing and not hung, instead of staring at a single `Installing` line and ~100s of silence before `Mounted`.

## Statement (Gherkin)

```gherkin
Given an AssetSpace whose tarball materializes more than 500 files
When mountAssetSpaceFiles writes the files via FileSystemPort.materialize
Then the materialize sub-phase emits a progress tick (processed, total) every 200 files written
And it does NOT emit a tick for the final file (the completion marker covers it)
And each tick carries the AssetSpace's total file count as the volume
And the activity log renders it as "Installing <label> <uid8>: <processed> of <total> files (<pct>%)"

Given an AssetSpace whose tarball materializes 500 or fewer files
When mountAssetSpaceFiles writes the files
Then no granular materialize file tick is emitted (the per-AS "Mounting … (i of N)" marker is the only signal)
```
