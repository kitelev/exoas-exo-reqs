# req(exo): `PreconditionEvaluator` token substitution is word-boundary-anchored (`$todayStart` not mangled)

## Job Story

When I author a visibility/availability precondition whose `sparqlAsk` happens to contain a `$token` that shares a prefix with a supported one — e.g. `$todayStart` (an executor / SubstitutionToken form, not a precondition token) — I want it to be left **literal** so the SPARQL parser rejects it loudly, rather than having its `$today` prefix silently rewritten into a malformed `"YYYY-MM-DD"^^xsd:dateStart` literal that produces a wrong-but-parseable ASK.

## Statement (Gherkin)

```gherkin
Scenario: an unknown $token sharing a prefix with a supported one is NOT partially substituted
  When PreconditionEvaluator.substituteVariables interpolates the query "$todayStart"
  Then the result is the literal "$todayStart" (unchanged — an unknown token)
  And it is NOT a mangled "YYYY-MM-DD"^^xsd:dateStart literal
  And a real "$now"-prefixed unknown token "$nowCompact" is likewise left literal

Scenario: every VALID token still substitutes (no regression from the word boundary)
  When PreconditionEvaluator.substituteVariables interpolates a query containing "$today" and "$now"
  Then "$today" becomes a "YYYY-MM-DD"^^xsd:date literal
  And "$now" becomes a "…"^^xsd:dateTime literal
  And no leftover "$today"/"$now" text remains
```

## Verification (plan)

**Binding class:** `Unit` — a leaf change confined to `PreconditionEvaluator.substituteVariables` (adding `\b` to the 8 calendar-date token regexes). Both the unknown-prefix boundary case and the valid-token non-regression case are exercised through the real `PreconditionEvaluator` (no mocks). Test title carries `@req:9bff6eb9-c8c2-4931-a93d-331334fc6e15`.

**Revert-verify** ([[integration-test-revert-verify]]): with the `\b` removed from `/\$today\b/g` (reverting to `/\$today/g`), `substituteVariables("$todayStart")` returns a mangled `"YYYY-MM-DD"^^xsd:dateStart` literal (≠ `"$todayStart"`) → **RED** (the test asserts the result equals the literal `"$todayStart"`); restored with `\b` → **GREEN**. The assertion is date-independent (it compares against the literal unchanged token), so it needs no fake clock; the valid-token non-regression case is covered by the existing `PreconditionEvaluator.todayLocalTz` suite (req `ecb90c06`).

> LOW-hardening follow-up preserved from the #3811 review (PDD-lossless). Family: #3806 (`$tomorrow`) → #3807/#3808 (`$today` SubstitutionToken) → #3809/#3810 (`$today` executor + form) → #3811/#3813 (5th `$today` surface, `PreconditionEvaluator` local basis, req `ecb90c06`). Design note (Homoiconicity Invariant): the precondition SPARQL and its `$token`s are vault-declared RDF; only the domain-agnostic text substitution lives in code — anchoring it with a word boundary makes unknown tokens fail loud rather than mangle.
