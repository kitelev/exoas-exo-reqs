# req(exo): TokenInvocation_parameter must stay a literal key so `$target.property(...)` defaults inherit from the target

## Context

RFC 727572d2 Universal Default Template: a `create_instance` grounding's PropertyDefault may use a `$target.property(<key>)` **TokenInvocation** to copy a value from the clicked target's frontmatter into the new instance. The canonical use is the Universal singleton's `exo__Asset_isDefinedBy = $target.property(exo__Asset_isDefinedBy)` — so every CLI/plugin-created instance inherits its ontology (co-location invariant) from the prototype it was created from.

## The bug (#3757)

`NoteToRDFConverter.valueToRDFObject` eagerly substitutes any `prefix__LocalName`-shaped string literal into its ontology class IRI. The `exocmd__TokenInvocation_parameter: exo__Asset_isDefinedBy` literal is therefore stored as `<https://exocortex.my/ontology/exo#Asset_isDefinedBy>` instead of the literal key. `CommandResolver` reads it via `getLiteralValue` (which returns IRI values verbatim), bakes the IRI into the parameterised marker, and the `targetProperty` resolver then does `ctx.targetFm["https://…#Asset_isDefinedBy"]` — a guaranteed miss (targetFm is keyed by `exo__Asset_isDefinedBy`) → returns `null` → the new instance is created with NO `exo__Asset_isDefinedBy`.

Affects **both** CLI `apply` and the plugin button (shared converter) — not a CLI-only parity gap. The issue's original premise (CLI never registers a `UniversalDefaultLoader`) is false: the CLI in-store fallback (`CommandResolver.findUniversalSingleton`) already loads the template (verified: all 6 PDs + 6 IRs reach the executor).

## Acceptance (revert-verify)

Through the real `convertVault → CommandResolver.loadCommand → GroundingExecutor.executeCreateInstance` path:
- **GREEN (with fix):** a `create_instance` applied to a prototype with `exo__Asset_isDefinedBy: "[[<onto>]]"` yields a new instance with `exo__Asset_isDefinedBy: "[[<onto>]]"`.
- **RED (without fix):** neutralise the converter bypass → the created instance loses `exo__Asset_isDefinedBy`.

The masking test must be fixed: the existing `CommandResolver.universalDefaultTemplate.test.ts` injects the `TokenInvocation_parameter` triple directly into the store, bypassing the converter (test-fixture-realism false-positive) — the new gate exercises the real converter.
