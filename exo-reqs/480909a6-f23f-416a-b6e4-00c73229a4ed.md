# req(exo): find --class <label> resolves the class via a find__Alias asset and emits the matching asset path

## Job Story

When I want to list assets of a given class from the command line, I want `find --class ems__Task` to look up the reusable "class" query fragment that lives as a find__Alias vault asset, bind my class label into it, and print the matching asset paths — so the search logic is homoiconic (editable as a vault asset) rather than hardcoded in the CLI.

## Statement (Gherkin)

```gherkin
Given a vault with a find__Alias asset labelled "class" carrying a SPARQL WHERE-fragment
And an ems__Task asset at a known path
When I run `find --vault <vault> --class ems__Task`
Then the CLI resolves the "class" find__Alias fragment, binds "ems__Task" into it, and runs the query
And stdout emits at least one line containing the matching asset's path
And no error is emitted
```
