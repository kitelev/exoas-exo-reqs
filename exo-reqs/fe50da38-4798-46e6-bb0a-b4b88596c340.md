# req(exo): the vault loader is all-or-nothing — a malformed asset produces zero triples and is reported as skipped (no partial leak)

## Job Story

When I index a vault that contains a malformed asset (empty required property, malformed Instance_class IRI), I want that asset to contribute **zero** triples and be reported as skipped — not leak a partial set of triples — so that one bad file can never corrupt the graph or crash downstream SPARQL/resolvers.

## Statement (Gherkin)

```gherkin
Given a vault containing a mix of well-formed assets and 9 malformed assets
  (each violating a required-property invariant — empty uid/label/status/parent,
   malformed Instance_class IRI, etc.; the BAD_FIXTURES set, numbered up to 11
   with 05/06 dropped since those properties became optional)
When I convert the vault via NoteToRDFConverter.convertVaultWithValidation
Then every malformed asset is reported in skippedFiles (skipped count = 9 = BAD_FIXTURES.length)
And no triple in the output has a subject derived from any malformed file
And the produced triples come exclusively from the well-formed subset (triple count > 0)
```
