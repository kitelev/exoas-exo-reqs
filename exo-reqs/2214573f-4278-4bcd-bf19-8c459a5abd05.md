# req(exo): vault check-runner — one-pass warm read + fail-loud on unregistered/unwired checks

## Job Story

When I run `Validate vault`, I want the engine to build its data ONCE (one warm pass, never per-file re-reads) and to never silently skip a check I enabled — an enabled check with no runner (or whose runner can't run) must surface as a loud error, not a green pass — so the report is trustworthy on desktop and mobile alike.

## Statement (Gherkin)

```gherkin
Given a VaultCheckRunner with the default registry and a reader that counts read() calls
When I run it with two enabled checks
Then reader.read() is invoked exactly once (one-pass / warm-cache contract)

Given an enabled check-id with no registered runner (or SHACL enabled with no triple-store wired)
When the runner runs
Then that check is reported with status "error" (fail-loud) and the overall report ok is false
```
