# req(exo): CLI can relabel an asset (set exo__Asset_label) and keep the aliases mirror in sync (set-label)

## Job Story

When I want to rename a task/project from the CLI (dogfooding mutation parity), I want `apply set-label <asset> --input '{"label":"New name"}'` to set `exo__Asset_label` and append the new label to `aliases` — so the rename is one homoiconic command, wikilinks-by-label keep resolving, and a label with YAML-special characters (e.g. a colon) does not silently break the file's frontmatter.

## Statement (Gherkin)

```gherkin
Given an asset with exo__Asset_label "Old Label" and aliases ["Old Label"]
When I run `apply set-label <asset> --input '{"label":"New Label"}'`
Then exo__Asset_label becomes New Label
  And aliases contains both "Old Label" and "New Label" (set-based dedup)
  And the body H1 is left untouched (freeform)
When the new label contains ': ' (e.g. "Meeting: Q3 review")
Then exo__Asset_label is written as a double-quoted YAML scalar and the file stays parseable
```
