# req(exo): CLI can relabel an asset (set exo__Asset_label) and keep the aliases mirror in sync (set-label)

## Job Story

When I want to rename a task/project from the CLI (dogfooding mutation parity), I want `apply set-label <asset> --input '{"label":"New name"}'` to set `exo__Asset_label`, keep the `aliases` mirror in sync as the SINGLE canonical label (not accumulating the old names), and stamp `exo__Asset_updatedAt` — so the rename is one homoiconic command, wikilinks-by-label keep resolving, the file does not collect stale aliases on every relabel, and a label with YAML-special characters (e.g. a colon) does not silently break the file's frontmatter.

## Statement (Gherkin)

```gherkin
Given an asset with exo__Asset_label "Old Label", aliases ["Old Label", "Some Stale Alias"] and exo__Asset_updatedAt "2020-01-01T00:00:00"
When I run `apply set-label <asset> --input '{"label":"New Label"}'`
Then exo__Asset_label becomes New Label
  And aliases contains ONLY "New Label" (single canonical mirror — the prior aliases are cleared, no accumulation)
  And exo__Asset_updatedAt is bumped to the current local timestamp (no longer 2020-01-01)
  And the body H1 is left untouched (freeform)
When the new label contains ': ' (e.g. "Meeting: Q3 review")
Then exo__Asset_label is written as a double-quoted YAML scalar and the file stays parseable
```
