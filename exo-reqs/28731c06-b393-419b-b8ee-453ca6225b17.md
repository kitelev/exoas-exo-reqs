# req(exo): exo-layout action buttons execute via structural exocmd commands (#3654 Part 2 / #3777)

## Job Story

When I author an `exo-layout` table with action buttons and I want a button to actually DO something (e.g. mark a task done, set a status) rather than just be shown/hidden, I want the button to reference a structural `exocmd__Command` and, on click, run that command's typed grounding through the same `CommandExecutionFlow` the structural inline command buttons already use — so the layout action has a real effect and the layout refreshes, without duplicating a parallel raw-SPARQL-UPDATE engine.

## Statement (Gherkin)

```gherkin
Scenario: clicking a layout action button that references a structural exocmd command runs it through CommandExecutionFlow
  Given an exo-layout table row for a real vault asset
  And an action button whose referenced command is a structural exocmd__Command (has exocmd__Command_grounding — a typed grounding)
  And the command's precondition (if any) holds for the row asset
  When I click the button
  Then the command is resolved via CommandResolver.loadCommand and run through CommandExecutionFlow.run
  And the run targets the row asset (targetIRI = the row's real store subject IRI, filePath = the row's asset path)
  And on success the layout block is refreshed (onComplete)

Scenario: a layout action referencing a raw-only or unresolvable command surfaces an honest notice, not a silent no-op
  Given an exo-layout action button whose referenced command is NOT a resolvable structural exocmd command
  When I click the button
  Then a user-facing notice explains no structural command could be executed
  And nothing is silently swallowed

Scenario: a structural command's own precondition gates the button (unified with Part 1)
  Given an exo-layout action button referencing a structural exocmd command that carries an exocmd__Command_precondition
  When the layout renders
  Then the button is shown only when that structural precondition evaluates true for the row asset (PreconditionEvaluator, the same path structural inline buttons use)
```
