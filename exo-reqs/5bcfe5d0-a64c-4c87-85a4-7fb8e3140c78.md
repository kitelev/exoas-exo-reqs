# req(exo): ExoSync keep-local of a cross-path (co-located rename) conflict consolidates and converges

## Job Story

When I keep my local version of a sync conflict for an asset that the remote has moved to a different (co-located) path, I want the resolution to consolidate the asset to a single remote path — so a subsequent sync sees no conflict and "keep local" actually sticks, instead of re-reporting the same conflict on every sync forever.

## Statement (Gherkin)

```gherkin
Given an asset with uid U exists LOCALLY at path A (e.g. tbank-efforts/X.md)
And the same uid U exists on the REMOTE at a DIFFERENT path B (e.g. og/X.md)
And the two versions' frontmatter diverged with no common sync base
And the conflict was quarantined and resolved by keep-local (queued in the deferred-push outbox)
When the next sync flushes the outbox
Then the chosen (local) content is pushed to path A on the remote
And the stale sibling copy at path B is deleted in the SAME commit (recoverable in git history — zero data loss)
And path A and path B are both unpinned
And a subsequent sync re-derives NO conflict (keep-local sticks)

Given the stale sibling copy at path B was itself edited on the remote after the offline resolution
When the next sync flushes the outbox
Then no remote path still holds the based-on blob, so nothing is deleted
And the queued push is dropped and the conflict re-surfaces for a fresh choice (zero data loss)
```
