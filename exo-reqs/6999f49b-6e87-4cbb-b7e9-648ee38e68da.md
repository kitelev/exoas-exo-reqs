# req(exo): audit ontology-imports flags a cross-ontology link as a violation unless the source ontology declares an exo__Ontology_imports closure covering the target

## Job Story

When I split my knowledge across many EKA AssetSpaces (ontologies) and an asset in one ontology links to an asset in another, I want `audit ontology-imports` to flag that cross-ontology link as a violation unless my source ontology explicitly declares the dependency via `exo__Ontology_imports`, so that the inter-ontology dependency graph stays explicit and auditable (and cannot grow hidden, cyclic coupling between AssetSpaces).

## Statement (Gherkin)

```gherkin
Given an asset in ontology A that links to an asset isDefinedBy ontology B
And ontology A does NOT declare exo__Ontology_imports covering B
When I run `audit ontology-imports` over the vault
Then the audit is not clean and reports one violation A→B
When ontology A declares exo__Ontology_imports including B
And I re-run the audit
Then the audit is clean (the cross-ontology link is counted valid)
When the import declaration is removed again
Then the violation A→B returns
```
