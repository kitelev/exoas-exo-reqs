# req(exo): pn__DailyNote Layout — independent show/hide of Actions / Tasks / Projects daily-efforts blocks (desktop + mobile, homoiconic)

## Job Story

When I open today's daily note, I want to turn the Actions, Tasks, and Projects sections on or off independently — set once as a sensible default for all daily notes, and override on a specific note when needed — without editing plugin code or digging in settings, and have it behave the same on my iPhone as on desktop.

## Statement (Gherkin)

```gherkin
Given a LayoutBlock with its visibility flag set to false
Then ExoLayoutRenderer does not render that block
And a sibling block with visibility true is rendered

Given a daily-efforts-by-class block on a pn__DailyNote
Then it selects the day's efforts (isEffortInDay) and partitions them by exo__Instance_class
And the «Actions» block shows ems__Action instances of the day
And the «Tasks» block shows the day's efforts EXCEPT Action and Project (so ems__Meeting and other Effort subclasses stay here — RL#1 inclusive carve-out)
And the «Projects» block shows ems__Project instances of the day
And the daily note's special navigation sections are still rendered (preserved through the block pipeline)

Given the default visibility (Tasks shown) and a day that has an ems__Meeting
Then the meeting still renders inside the «Tasks» block
# RL#1 — the primary falsifier of "0 regressions"

Given a per-note frontmatter visibility override AND an exo__Layout default AND the built-in default
Then the effective visibility resolves as: frontmatter override > exo__Layout default > built-in default

Given override values arriving as true / false / "true" / "false" / absent / null / a partial key set
Then each coerces deterministically to a boolean visibility with no unexpected block appearing or disappearing

Given a pn__DailyNote with NO visibility configuration anywhere
Then Actions is shown, Tasks is shown, and Projects is hidden
# VL#3 — out-of-the-box default

Given ems__Action declared as a subclass of ems__Effort
Then SHACL validation is clean
And existing Effort-wide queries/renderers show no regression

Given the daily-note Layout read-path
Then it uses only Obsidian-core APIs via vault.adapter with no Platform gate
And it renders identically on desktop and on mobile (Desktop↔Mobile Command Parity invariant)

Given block visibility is configured
Then it lives in the exo__Layout RDF asset and per-note frontmatter — never in plugin settings (Homoiconicity Invariant)
```
