# req(exo): applying a workflow_transition command advances ems__Effort_status to the next workflow state and runs its postActions

## Job Story

When I apply a workflow command to an effort (e.g. «Start task»), I want its `ems__Effort_status` advanced to the correct next state in the workflow AND any postActions of that transition executed, so a single command moves the effort forward consistently instead of me hand-editing the status frontmatter.

## Statement (Gherkin)

```gherkin
Given a task in status Backlog whose class has a workflow with a forward Backlog → Doing transition
When I apply the forward workflow_transition command to it
Then ems__Effort_status is mutated to Doing (UID-form wikilink) and is no longer Backlog
And the transition's postAction sentinel (ems__Effort_postActionMarker: ran) is written
```
