# req(exo): the effort list prioritizes not-done tasks over done tasks

## Job Story

When I look at a list of efforts, I want the tasks I still have to do to appear before the ones already done, so completed work sinks and my open work stays visible.

## Statement (Gherkin)

```gherkin
Given a not-done task A and a done task B (same trashed-state)
When the effort list is sorted by priority
Then task A (not done) is ordered before task B (done)
```
