# req(exo): `cli create --property` accumulates repeated keys into a YAML array (multi-value)

## Job Story

When I create a vault asset with `exocortex-cli create` in ONE step and the asset has a multi-value property (a `exo__Asset_relates` with several links, an `exo__Instance_class` with several classes, …), I want to pass the same `--property key=value` flag multiple times and have the values collected into a YAML array — so I never have to follow up with a manual `Edit`, and "vault assets are created only through the CLI" stays achievable.

## Statement (Gherkin)

```gherkin
Given `exocortex-cli create` for any class
When I pass `--property exo__Asset_relates=[[A]] --property exo__Asset_relates=[[B]]`
Then the produced frontmatter `exo__Asset_relates` is a YAML array of two elements ([[A]], [[B]])
And no value is lost (last-wins is gone)

Given `exocortex-cli create` for any class
When I pass a single `--property exo__Asset_relates=[[A]]`
Then the produced frontmatter `exo__Asset_relates` is the scalar string "[[A]]" (back-compat)

Given any property key (not only relates)
When I pass that key more than once via `--property`
Then the values accumulate into a YAML array on that key
```

## Verification

**Bound (integration):** `@req:cd2c3f18-b798-4084-899d-38e2f438a2cc` — a production-shape integration test drives the real `createCommand()` end-to-end against a temp vault (commander `collect` → `parseProperties` → core `GenericAssetCreationService` → written frontmatter) and asserts the multi-value array + single-value scalar back-compat. Revert-verified: restoring the last-wins `parseProperties` reduce turns the multi-value assertion RED. al-cli-3759, 2026-06-28.

## Scope note (flow-array, Repro 2)

Issue #3759 Repro 2 (`--property 'exo__Asset_relates=["[[A]]","[[B]]"]'` → escaped literal) is **deliberately out of scope** of this requirement. Parsing a flow-array literal inside one `--property` introduces value/array ambiguity (is `[...]` a literal value or a list?) that the AC does not require. The supported multi-value path is the repeatable flag; the flag help documents this. A single `--property` value is always taken literally.
