# req(exo): in-plugin «Deduplicate uids» on desktop AND mobile — shared platform-free dedup core

## Job Story

When a sync or conflict-resolution message tells me to "run `exosync dedup-uids`" — and I am on my iPhone, where there is no CLI — I want an in-plugin «Exocortex: Deduplicate uids» command that reports which `exo__Asset_uid`s are duplicated (and on which files) and, on my confirmation, reassigns a fresh uuid to every duplicate but the first (without ever renaming a file) — so the remediation the plugin points me at is actionable on the same device, not a desktop-only / CLI-only escape hatch.

## Statement (Gherkin)

```gherkin
Given the plugin is loaded (on desktop OR mobile)
Then the «Deduplicate uids» command is registered with the stable id `exosync-dedup-uids` on both platforms (no Platform.isMobile gate)

Given a vault with two or more files declaring the SAME exo__Asset_uid
When I run «Deduplicate uids»
Then I am first shown a report listing every duplicated uid and the paths declaring it

Given that report and my confirmation to proceed
When the fix runs over vault.adapter (no Node fs)
Then the lexicographically-first file in each duplicate group keeps its uid
And every other file in the group gets a fresh uuid written to its frontmatter exo__Asset_uid
And no file is renamed
And re-running the command reports no duplicate uids

Given the same vault but I cancel at the confirmation
Then no file is modified
```

## Notes

- Mirrors the CLI `runDedupUids` `--fix` semantics (NOT the `--auto` zero-loss resolver). The dedup core (`findDuplicateUidGroups` / `planDuplicateUidFix` / `rewriteAssetUid`) is extracted to `@kitelev/exocortex-core` so the CLI and plugin reassign uids identically; the CLI is refactored onto it with no behaviour change.
- Desktop↔Mobile Command Parity invariant (CLAUDE.md): the command is registered unconditionally and the fix is driven over `vault.adapter`, never Node `fs`.
