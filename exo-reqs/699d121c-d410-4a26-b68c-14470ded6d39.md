# req(exo): create emits a property in scalar vs YAML-array form according to its declared SHACL cardinality

## Job Story

When I create an asset whose property is declared Multiple-cardinality (e.g. `ems__Effort_relatesTo`), I want that property emitted as a YAML array so Obsidian and the RDF parser both treat it as a multi-value field — while Single-cardinality and undeclared properties stay scalar — so the created frontmatter is shape-correct and round-trips without cardinality drift.

## Statement (Gherkin)

```gherkin
Given a vault whose SHACL shapes declare ems__Effort_relatesTo as Multiple-cardinality (via a UID-form exo__Property_cardinality wikilink)
When I create an asset setting that property to a single value
Then the property is emitted as a YAML array (one element)

Given a property declared Single-cardinality (ems__Effort_status) or with no cardinality declared
When I create an asset setting that property
Then the property is emitted scalar (inline value, not an array)
```
