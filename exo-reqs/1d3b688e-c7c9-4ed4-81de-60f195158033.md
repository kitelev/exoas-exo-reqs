# req(exo): apply --seed + --frozen-clock makes asset creation deterministic — two runs yield a byte-identical file

## Job Story

When I want to snapshot- or golden-file-test a command that creates an asset, I want `apply --seed <uuid> --frozen-clock <iso>` to make the generated UID and createdAt timestamp deterministic, so that re-running the same command produces a byte-identical file and the test can assert exact output instead of masking the live UID/clock.

## Statement (Gherkin)

```gherkin
Given a command that creates an asset
When I run `apply <command> <target> --seed <uuid> --frozen-clock <iso>` twice with the same seed and frozen clock
Then both runs derive the UID from the seeded generator and the createdAt from the frozen clock
And the two created files are byte-identical (same filename / uid and same content)
```

## Verification

(seededUidGenerator(options.seed), liveUidGenerator()) : liveUidGenerator()` — the seeded generator is created but its result discarded, a live non-deterministic one is used) makes `apply-determinism.integration.test.ts "two runs with identical --seed and --frozen-clock yield byte-identical created file"` go **RED** (the two runs produce different UID filenames, e.g. `b84f6ba0….md` vs `d36a381d….md`, so the byte-identical assertion fails); restored → **GREEN** (2026-06-21, al-reqmgmt-ramp2, origin/main `4b940b33`).
