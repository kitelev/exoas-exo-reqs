# req(exo): validate schema --shapes-mode flags a sh:maxCount violation when a Single-cardinality property carries 2 values

## Job Story

When I run schema validation over the vault before committing, I want a property declared with Single cardinality (sh:maxCount=1) that accidentally carries two values to be reported as a hard violation, so that cardinality drift cannot slip into the graph and break downstream resolvers/queries that assume scalars.

## Statement (Gherkin)

```gherkin
Given an asset whose ems__Task_name property is declared Single cardinality (sh:maxCount=1)
And that asset carries 2 values for ems__Task_name
When I run `validate schema --shapes-mode` over the vault
Then the validator reports a violation focused on that asset and the ems__Task_name path
And the violation severity is sh:Violation
And the violation message contains "sh:maxCount violation"
```
