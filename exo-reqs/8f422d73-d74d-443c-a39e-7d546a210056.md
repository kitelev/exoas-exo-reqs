# req(exo): SPARQL OPTIONAL uses LEFT JOIN semantics — every left solution is preserved, optional variables bound only when the optional pattern matches (#2208)

## Job Story

When I write a SPARQL query (in a layout or a self-audit) that lists assets with an OPTIONAL related entity, I want EVERY asset returned — with the optional variable filled in only where the relation exists — so that partially-populated assets (a Task with no Project yet, a Person with no email) are not silently dropped from my views and counts.

## Statement (Gherkin)

```gherkin
Given a triple store with tasks, only some of which link to a project
And a query that matches tasks and OPTIONALly binds their project
When the query is executed
Then every task is returned
And ?project is bound for tasks that have a project
And ?project is unbound (not filtered out) for tasks that have none
```
