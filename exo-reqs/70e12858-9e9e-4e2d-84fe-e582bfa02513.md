# req(exo): under SPARQL STRICT mode, a label-form (unresolvable) profile _imports throws instead of silently under-counting the imports closure

## Job Story

When a profile declares its exo__Profile_imports with a label-form wikilink that doesn't resolve, I want STRICT mode to FAIL LOUD (throw) rather than quietly return a partial imports closure that looks successful — so that a 2/3 recall loss in profile composition is surfaced in CI instead of silently shipping a wrong effective AssetSpace set (EV7 / issue #3427).

## Statement (Gherkin)

```gherkin
Given a profile P-root whose exo__Profile_imports value is label-form
  (an unresolvable wikilink stored as a bare literal — EV7 mandate violation)
And a parent profile that contributes two further AssetSpaces to the closure
When I run the CQ2 imports-closure query (_includes ∪ transitive(_imports*))
  with EXOCORTEX_SPARQL_STRICT=1
Then the query THROWS (literal in subject position) rather than returning rows
And the throw prevents a silent 2/3 recall loss
  (without STRICT it would return only the directly-declared AssetSpace)
```
