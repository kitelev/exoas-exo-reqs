# req(exo): ExoSync 3-way merge combines non-overlapping edits without conflict or silent overwrite

## Job Story

When the same asset is edited on two devices in non-overlapping ways (different frontmatter keys, or the remote advancing on disjoint files since I mounted), I want ExoSync to merge both sides cleanly rather than conflict spuriously or silently overwrite one side, so I never lose work and only get a conflict when the edits genuinely collide.

## Statement (Gherkin)

```gherkin
Given a 3-way merge base where local changed key K1 and remote changed a different key K2 (no overlap)
When ExoSync's structured merger merges the asset
Then the merge succeeds with K1 = local's value and K2 = remote's value (no conflict)

Given a first sync where the remote advanced on files disjoint from the local working-tree edits
When the SyncEngine syncs
Then it clean 3-way merges (pulls the remote-only change, pushes the local-only edits)
And it never silently overwrites the remote change with the stale local copy
```
