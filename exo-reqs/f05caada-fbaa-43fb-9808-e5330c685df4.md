# req(exo): "Clean Properties" command visible only when the asset has empty properties

## Job Story

When I open an asset that has **no empty frontmatter properties**, I do **not** want to
see the "Clean Properties" maintenance button, so that the inline command bar stays
uncluttered and I am not offered a destructive no-op. When an asset **does** carry empty
properties, the button should appear so I can clean them.

## Acceptance (Gherkin)

```gherkin
Feature: Clean Properties command is gated by the presence of empty frontmatter properties

  Background:
    Given the homoiconic "Clean Properties" exocmd command (asset 0da175e1)
    And its precondition declares host function "hasObsoleteProperties"
    And the "hasObsoleteProperties" host function is registered in code
      (a factory that reads the target's frontmatter and returns hasEmptyProperties(metadata))

  Scenario: Asset WITH an empty property → command available
    Given a target asset whose frontmatter contains at least one empty value
      (null, empty string, empty array, or empty object)
    When the "Clean Properties" command precondition is evaluated for that asset
    Then the precondition passes
    And the inline button / palette entry is shown
    And `apply clean-properties <asset>` proceeds (and removes the empty property)

  Scenario: Asset with NO empty property → command hidden
    Given a target asset whose every frontmatter value is non-empty
    When the "Clean Properties" command precondition is evaluated for that asset
    Then the precondition fails
    And the inline button / palette entry is hidden
    And `apply clean-properties <asset>` is blocked with "Precondition not satisfied"

  Scenario: Regression guard — the fail-open bug
    Given the "hasObsoleteProperties" host function is NOT registered
    When the "Clean Properties" command precondition is evaluated for a clean asset
    Then evaluateHostFunction returns true (fail-open) and the button wrongly appears
    # This is the pre-fix behavior the registration removes (revert-verify RED direction).
```

## Notes

- The host-function NAME `hasObsoleteProperties` is a legacy identifier baked into the
  vault precondition asset (79a7fb4b). Keeping it (rather than renaming to
  `hasEmptyProperties`) means the fix reaches the user on a plain plugin update with **no**
  re-sync of the `exoas-exocmd` assetspace. The behaviour it computes is "has empty
  frontmatter values".
- Mirrors the `isInWrongFolder` / "Repair Folder" host-function wiring
  (`createIsInWrongFolderHostFunction`, Issue #3302).
