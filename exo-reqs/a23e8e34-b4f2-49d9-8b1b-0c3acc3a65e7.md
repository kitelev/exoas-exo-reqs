# req(exo): applying a leaf profile materializes its transitive dependsOn closure from the central registry

## Job Story

When I apply a profile that declares only the leaf assetspaces I care about, I want every assetspace they transitively depend on (the EKA registry's `dependsOn` DAG) to be resolved and materialized too, so I get a working effective set without manually enumerating the whole dependency chain.

## Statement (Gherkin)

```gherkin
Given a leaf profile in a central-registry vault whose declared assetspace depends (transitively)
  on further assetspaces via exo__AssetSpace_dependsOn
When I apply that profile
Then the resolver returns "engaged" (not degraded/refuse)
And the effective set materialized is the full transitive dependsOn closure (not just the literal _includes)
```
