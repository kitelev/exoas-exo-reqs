# req(exo): ExoSync quarantine resolve keep-remote preserves the discarded local in a .txt backup (zero data loss)

## Job Story

When I resolve a sync conflict by keeping the remote version (discarding my local edit), I want my discarded local content preserved byte-for-byte in a recoverable backup that never re-syncs and never pollutes the knowledge graph — so resolving a conflict can never silently lose the work I was discarding.

## Statement (Gherkin)

```gherkin
Given a quarantined 3-way conflict on PATH where local, base, and remote all differ
When I resolve it with keep-remote
Then the remote content is written to disk at PATH
And the discarded local content is preserved byte-for-byte at PATH.conflict.local.txt
And that backup is non-syncable (a .txt, .conflict. infix — excluded from both sync predicates and the triple store)
And the path is unpinned (the conflict is cleared)
```
