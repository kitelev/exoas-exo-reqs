# req(exo): read-view Relations empty-state → «+ Add relation» affordance (desktop + mobile)

## Job Story

When I open a freshly-created asset that has no relations yet, I want a visible «+ Add relation» control right in the empty Relations section so I can add my first relation from the read view — without having to discover the command palette name «Edit properties» — on my iPhone just as on desktop.

## Statement (Gherkin)

```gherkin
Given the read-view Asset Relations block of an asset with ZERO relations, rendered with the asset's currentFile
Then the empty-state section renders a «+ Add relation» affordance (.exocortex-relations-empty-add) under the muted "No related assets yet" text

Given the empty-state «+ Add relation» affordance
When I activate it
Then the current asset's relations editor opens
And it opens via the registered `exocortex:edit-properties` command (one source of truth — the same flow as the ≥1-branch «Edit relations» affordance and Cmd+P)

Given the plugin is loaded on mobile
Then the affordance is wired with no platform gate, using only Obsidian-core APIs (no Node fs)
And activating it opens the editor exactly as on desktop (Desktop↔Mobile Command Parity invariant)

Given a legacy caller that does not thread the asset's currentFile into RelationsRenderer.render
Then no «+ Add relation» affordance is rendered in the empty-state (additive wiring — zero read-view regression)

Given an asset with ≥1 relation
Then the empty-state affordance is not rendered and the existing block-level «Edit relations» affordance path is unaffected
```

## Verification

**Bound (component + unit):** `@req:57fc5804-2c67-4fb6-ba3d-5bac562da7c1` — the RelationsRenderer empty-state test asserts the `.exocortex-relations-empty-add` button is present iff `currentFile` is provided, that activating it calls `app.commands.executeCommandById('exocortex:edit-properties')`, and that the ≥1-relation path is unaffected. Revert-verified (removing the empty-state affordance block → case (a) RED, (b)+(c) GREEN, compiles cleanly; restore → all GREEN). ## Notes

- **Why it complements cbb982a3:** req cbb982a3 added the block-level «Edit relations» affordance, but that affordance lives in the ≥1-relation branch (the React relations table). The zero-relations empty-state has no table to host it, so the read-view editor entry point was still missing for exactly the new-user / first-run scenario. This requirement closes that empty-state gap with the SAME editor command (no divergence).
- **Placement (AC #3):** the affordance lives INSIDE the empty-state block (under the muted text), not a section-header button — there is no Relations table in the empty-state, so the affordance sits where the user is already looking.
- **Priority LOW (P3):** a Cmd+P workaround exists; this is onboarding polish, not a blocker. Reversible at review.
