# req(exo): rename-to-uid collapses inbound wikilinks to bare [[uid]] (strip alias, preserve anchor, skip code/embeds)

## Job Story

When I rename an asset to its UID-form filename, I want every inbound wikilink across my vault rewritten to the new bare `[[uid]]` target — alias dropped, navigation anchors kept, and code/embeds left untouched — so the rename never silently breaks references and the vault stays UID-canon-consistent.

## Statement (Gherkin)

```gherkin
Given an asset "Choco" referenced across the vault as [[Choco]], [[Choco|Шоколад]], [[Choco#Ingredients]] and [[Choco#Ingredients|See]]
And the same name appears inside fenced code, inline code, an embed ![[Choco]], an already-UID-form link, and an unrelated [[Banana]] link
When I run `rename-to-uid` on the Choco file
Then the bare and aliased inbound links collapse to [[<uid>]] (alias dropped)
And heading-anchored links become [[<uid>#Ingredients]] (anchor preserved)
And fenced code, inline code, the embed, the already-UID-form link, and the unrelated link are left unchanged
```
