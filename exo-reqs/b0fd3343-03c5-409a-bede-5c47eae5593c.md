# req(exo): ExoSync SHACL merge-gate blocks a merged asset that violates an intrinsic constraint (never ships an invalid auto-merge)

## Job Story

When ExoSync auto-merges two divergent edits of the same asset, I want the merged result to be checked against the schema's intrinsic constraints (required properties, cardinality, datatype) BEFORE it ships — and refused (routed to quarantine) if it would be invalid — so an automatic merge can never silently produce and propagate a schema-broken asset.

## Statement (Gherkin)

```gherkin
Given a merged candidate asset typed as ems__Task
  And the SHACL shapes require ems__Effort_status with minCount 1 (sh:Violation)
  And the merged candidate is missing ems__Effort_status
When the ExoSync merge-gate checks the merged asset
Then the gate verdict is ok:false with a minCount violation
And the invalid merge is refused (routed to quarantine), never shipped
```
