# req(exo): an instance inherits its prototype's properties it does not own

## Job Story

When I create an instance from a prototype, I want it to carry the prototype's properties I did not set myself, so I do not re-enter shared defaults every time.

## Statement (Gherkin)

```gherkin
Given an instance asset declares exo__Asset_prototype pointing at a prototype
  that has a ems__Effort_area, and the instance has no ems__Effort_area of its own
When the prototype chain is materialized
Then the instance inherits the prototype's ems__Effort_area
  (and own values, when present, override the inherited one)
```
