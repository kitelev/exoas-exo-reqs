# req(exo): applying a narrower profile unmounts assetspaces outside the effective set (floor preserved)

## Job Story

When I apply a profile that no longer includes an assetspace I previously had mounted, I want that assetspace's files removed from my vault (and its `.gitmodules` stanza stripped) while the TS-floor stays put, so my mount-state matches the profile and I never accidentally brick the engine.

## Statement (Gherkin)

```gherkin
Given an assetspace (testlib) is materialized on disk and a narrower profile is selected
  that does not include it (but still declares the TS-floor)
When I apply the narrower profile
Then the testlib assetspace folder is removed and its .gitmodules stanza is stripped
And the TS-floor assetspaces ($exo, $exocmd, $shared-identities) remain materialized
And the testlib descriptor survives (in the registry) for switch-back
```
