# req(exo): bootstrapping an empty git vault materialises the exo TS-floor at the Maven path and writes its single .gitmodules entry

## Job Story

When I bootstrap a fresh empty git vault, I want the exo engine floor materialised at the canonical Maven path and recorded as a single `.gitmodules` entry, so a later `apply-profile` recognises it as already-mounted (no double-mount) and the vault has exactly the minimal exo-only clean start — exocmd and everything else is added later.

## Statement (Gherkin)

```gherkin
Given an empty git vault and the exo TS-floor repo URL
When I run «Exocortex: Bootstrap vault»
Then the exo AssetSpace is materialised at the Maven path assetspaces/<owner>/<repo>
And `.gitmodules` contains exactly the one exo entry at that Maven path
And exocmd is NOT materialised by bootstrap (added later via «Add AssetSpace» or a profile)
And a "Bootstrap complete" notice is shown
```
